What's new in OnePatch. We post notable changes here as they ship.

<Update label="June 30, 2026" tags={["Agent"]}>
  ## OnePatch speaks MCP, both ways

  Your workspace now exposes an MCP server your coding agent can connect to — query your live telemetry and read your chats from Claude Code, Cursor, or any MCP client. And the OnePatch agent can now use tools you bring: connect any OAuth-protected MCP server from the Integrations screen. See [MCP](/mcp).
</Update>

<Update label="June 22, 2026" tags={["Slack"]}>
  ## Talk to the agent from Slack

  DM the OnePatch bot, or @-mention it in a channel and it replies in a thread. Each conversation is a real chat that also shows up in the app, with the sender's name. Incident alerts now link straight to the incident chat, and you can rename the default alerts channel. See [Incidents and Slack](/incidents-and-slack).
</Update>

<Update label="June 10, 2026" tags={["Docs"]}>
  ## Documentation and public changelog

  OnePatch now has public documentation at [docs.onepatch.dev](https://docs.onepatch.dev), including a one-stop [getting-started guide](/getting-started) and reference for monitors, dashboards, and incidents. This changelog is where we will post notable changes from here on.

  The docs are written to be read by an AI agent as well as a person — the full text is available at [docs.onepatch.dev/llms-full.txt](https://docs.onepatch.dev/llms-full.txt).
</Update>
