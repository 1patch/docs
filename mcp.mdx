---
title: MCP
description: Use OnePatch from your own tools, and give the agent yours.
---

OnePatch speaks MCP — the Model Context Protocol — in both directions. Your coding agent can read your workspace's telemetry and chats, and your workspace's agent can use tools from MCP servers you connect.

## Query OnePatch from your coding agent

Your workspace exposes an MCP server at `https://app.onepatch.dev/mcp`. Add it to Claude Code, Cursor, or any other MCP client, and sign in with your OnePatch account when prompted — everything it returns is scoped to your organization.

For Claude Code:

```sh
claude mcp add --transport http onepatch https://app.onepatch.dev/mcp
```

The server exposes three tools:

- **`query_otel`** — run a read-only SQL query against your telemetry (`otel.spans`, `otel.metrics`, `otel.logs`).
- **`list_chats`** — list the chats in your workspace.
- **`read_chat`** — read one chat's transcript.

This lets a coding agent working in your repository answer questions from live production data — "did errors move after my last deploy?" — or read what the OnePatch agent found during an incident.

## Connect your own MCP servers

The agent can also use tools you bring. On the **Integrations** screen, add an MCP server's URL under **MCP servers**, then click **Connect** to authorize it. From then on, the agent has that server's tools in every chat you start — so it can reach the systems your team already uses.

Servers you add are visible to your whole organization, but each member authorizes their own access. The agent only uses a server on your behalf after you have connected it.
